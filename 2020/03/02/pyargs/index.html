Python中的*args和**kwargs

无标签
发布日期:   2020-03-02
更新日期:   2020-03-02
文章字数:   894

经常使用Python的话，相信你在函数参数里已经频繁遇见*args和**kwargs这两个符号。最常见的场景比如调包侠们经常要查看某个库里面某个接口接受那些参数，一连串的参数后面经常以这两个参数结尾。下面简单介绍下参数这种写法到底是啥意思。
 
先看个求和函数的例子：

def all_sum(a, b):
    return sum(a + b)

输入 

all_sum(3, 5)

结果 

8

如果这个函数是你写给别人用的API，而人家并不知道这里只能输入两个参数，于是他输入了： 

all_sum(3, 5, 0, -6)

那么必然会报错： 

-----------------------------------------------------------
TypeError                          Traceback (most recent call last)
<ipython-input-97-70e77b81c288> in <module>
----> 1 all_sum(3, 5, -0, -6)

TypeError: all_sum() takes 2 positional arguments but 4 were given

all_sum()只能接收2个位置参数，但是你给了4个。
 

这时候Python提供了这样一种运算符，我们将all_sum()函数改进一下： 

def all_sum_v2(*args):
    print('arg type: {)'.format(type(args)))
    print('args: '.format(args))
    print(agrs[3])
    return sum(args)

我们再输入任意个参数：

all_sum_v2(3, 5, 1, -4, 6)
type<class 'tuple'>
args: (3, 5, 1, -4, 6)
-4
11

为了搞清楚args到底是啥，我特意在其中多加了3个print。输出第一行说明args的类型是一个tuple，第二、三行看出args就是你所输入的参数组成的一个tuple。
这说明在定义函数时，*args将你的所有输入组织成一个tuple。
 
我们再来第一个版本的all_sum()函数，假如我现在的输入是一个列表[3， 5]，如果我直接把列表作为参数： 

num_list = [3, 5]

all_sum(num_list)

这时会报错： 

--------------------------------------------------------
TypeError                         Traceback (most recent call last)
<ipython-input-98-52a2b35e240d> in <module>
----> 1 all_sum(num_list)

TypeError: all_sum() missing 1 required positional argument: 'b'

结果并不是我们所期望的去求列表里面元素的和。这时我们可以这样写： 

all_sum(*num_list)

输出

8

所以，在定义函数时*将所有参数打包成一个tuple; 而参数被函数调用时，*将list或tuple解包成单个元素。据说在其他语言里这个星号叫splat运算符，感兴趣的童鞋可以自行觅食。
 
搞懂了*args，**kwargs就不难理解了。因为只要你稍微熟悉Python，应该就知道位置参数和关键字参数；不过这儿*args一个星号和*kwargs两个星号还是有一些区别的，继续看例子。 

def add_age_by_name(name1, name2):
    return name1 + name2

add_age_by_name(name1=22, name2=30)

得到输出： 

52

如果你输入超过2个关键字参数，比如add_age_by_name(name1=22, name2=30,name3=7)，那毫无疑问会报错。
不妨先把参数写成一个字典： 

parameters = {
    'name1': 22,
    'name2': 30
}

然后我们这样传入： 

add_age_by_name(**parameters)

得到同样正确的输出： 

52

可以看到**将字典中的key-value对解包成函数的参数名-参数值对了。那么，我们类比刚刚的*args将add_age_by_name()作改进： 

def add_age_by_name(**info):
    return sum([value for key, value in info.items()])

add_age_by_name(name1=20, name2=30, name3=18, name_one_more=5)

得到输出： 

73

同样总结一下，定义函数时，**将输入的关键字参数的参数名-参数值组织成一个字典；而被函数调用时，**将参数字典中的key-value对解包成函数的参数名-参数值对。
 
这样，*args和**kwargs就讲完了。
